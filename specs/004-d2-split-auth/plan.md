# Implementation Plan: D2 Split Authentication

**Branch**: `004-d2-split-auth` | **Date**: 2026-06-19 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/004-d2-split-auth/spec.md`

---

## Summary

Implement a bifurcated authentication flow for D2 that routes users through either a
traditional password form (`login_form` role) or a passwordless magic-link flow
(`auto_login` role) based on the role assigned in Keycloak. All logic lives entirely
in Keycloak's authentication flow configuration and a third-party magic-link provider
extension. D2's Django application requires only a session-alignment fix (`SESSION_COOKIE_AGE`
+ `OIDC_RENEW_ID_TOKEN_EXPIRY_SECONDS`). D1 is not touched.

- Mailtrap sandbox SMTP: `sandbox.smtp.mailtrap.io:587` with StartTLS

**Scale/Scope**: Dev/test environment. Two test users (one per role). No performance targets beyond normal Keycloak OIDC throughput.

## Detailed Change Specifications

### 1. `keycloak/Dockerfile` (NEW)

Two-stage build — Alpine downloads the JAR, Keycloak image copies it (Keycloak 24 base has
no `wget` or `curl`):

```dockerfile
ARG MAGIC_LINK_VERSION=0.26

FROM alpine AS downloader
ARG MAGIC_LINK_VERSION
RUN wget -q \
    "https://repo1.maven.org/maven2/io/phasetwo/keycloak/keycloak-magic-link/${MAGIC_LINK_VERSION}/keycloak-magic-link-${MAGIC_LINK_VERSION}.jar" \
    -O /keycloak-magic-link.jar

FROM quay.io/keycloak/keycloak:24.0
COPY --from=downloader /keycloak-magic-link.jar /opt/keycloak/providers/keycloak-magic-link.jar
```

> **JAR version**: `0.26` from Maven Central (`io.phasetwo.keycloak:keycloak-magic-link:0.26`).
> This is the KC 24.0-compatible release. Version `0.72` (KC 26) is incompatible — causes
> filesystem error on startup. See `research.md §1` for full rationale.

### 3. `keycloak/realm-export.json`

Five targeted changes (see `research.md` and `data-model.md` for full JSON fragments):

**3a. SMTP configuration** — add to realm root:
```json
"smtpServer": {
  "host": "${SMTP_HOST}",
  "port": "${SMTP_PORT}",
  "from": "${SMTP_FROM}",
  "auth": "true",
  "user": "${SMTP_USER}",
  "password": "${SMTP_PASSWORD}",
  "starttls": "true",
  "ssl": "false"
}
```
SMTP credentials are injected via environment variables, never hardcoded.
Add the corresponding variables to `.env` and to the `keycloak:` service environment in `docker-compose.yml`.

**3b. New client roles** — add to `d2-client` roles array:
```json
{ "name": "login_form",  "description": "Password-based authentication flow" },
{ "name": "auto_login",  "description": "Magic-link passwordless authentication flow" }
```

**3c. Redesigned D2 Browser Flow** — replace the `d2-client-forms` subflow content.
Currently it uses `auth-username-password-form` (combined form). Replace with:
- `auth-username-form` (identity-first: username only)
- `d2-password-branch` (CONDITIONAL subflow)
  - `conditional-user-role` → checks `d2-client.login_form` (negate: false)
  - `auth-password-form` (REQUIRED)
- `d2-magic-link-branch` (CONDITIONAL subflow)
  - `conditional-user-role` → checks `d2-client.auto_login` (negate: false)
  - `magic-link` authenticator from Phase Two JAR (REQUIRED)

Full JSON in `data-model.md`.

**3d. Magic-link authenticatorConfig** — add:
```json
{
  "alias": "d2-magic-link-cfg",
  "config": {
    "expirationInSeconds": "3600",
    "singleUse": "true",
    "sendEmailOnError": "false",
    "redirectUriTemplate": "http://localhost:8002/oidc/callback/"
  }
}
```

**3e. d2-client session override** — add to d2-client `attributes`:
```json
"client.session.max.lifespan": "28800",
"client.session.idle.timeout": "28800"
```
These are the Keycloak 24 attribute keys for per-client session overrides. See `research.md`.

### 4. `d2/config/settings/base.py`

Add three settings after the existing `SESSION_COOKIE_SAMESITE` line:

```python
SESSION_COOKIE_AGE = 28800                    # 8 hours — matches Keycloak client session max
OIDC_RENEW_ID_TOKEN_EXPIRY_SECONDS = 3600     # check with Keycloak every 1 hour
OIDC_USE_NONCE = False                        # required for cross-context magic link (see §7)
```

**Why these values**:
- `SESSION_COOKIE_AGE = 28800`: Prevents Django's session cookie from expiring before
  Keycloak's session (Django default is 2 weeks, which is too long).
- `OIDC_RENEW_ID_TOKEN_EXPIRY_SECONDS = 3600`: The `SessionRefresh` middleware will
  silently renew with Keycloak every hour. Within the 8h window, Keycloak silently
  re-issues the token. After 8h, Keycloak rejects renewal → full re-login triggered.
  User is logged out within 1 hour of the 8h mark (worst case: 9h, best case: 8h).
- `OIDC_USE_NONCE = False`: Required so that when `MagicLinkCallbackView` injects a
  synthetic `"nonce": None` state entry, the OIDC backend does not compare it against
  the real nonce in the ID token. See `research.md §7`.

> See `research.md §4` for the session alignment trade-off and `research.md §7` for the
> nonce trade-off.

### 5. `d2/accounts/views.py` — MagicLinkCallbackView (NEW)

```python
from mozilla_django_oidc.views import OIDCAuthenticationCallbackView

class MagicLinkCallbackView(OIDCAuthenticationCallbackView):
    def get(self, request):
        state = request.GET.get("state", "")
        if state and (
            "oidc_states" not in request.session
            or state not in request.session.get("oidc_states", {})
        ):
            existing = request.session.get("oidc_states", {})
            existing[state] = {"nonce": None, "code_verifier": None}
            request.session["oidc_states"] = existing
            request.session.modified = True
        return super().get(request)
```

This handles the case where the magic link is opened in a different browser context.
See `research.md §7` for the full problem description and rationale.

### 6. `d2/config/urls.py` — Override `/oidc/callback/`

```python
from accounts.views import MagicLinkCallbackView

urlpatterns = [
    path('oidc/callback/', MagicLinkCallbackView.as_view(), name='oidc_authentication_callback'),
    path('oidc/', include('mozilla_django_oidc.urls')),
    path('', include('accounts.urls')),
    path('', include('portal.urls')),
]
```

The custom path must appear **before** `include('mozilla_django_oidc.urls')` so Django
resolves `/oidc/callback/` to the custom view. All other `/oidc/` paths (`/oidc/authenticate/`,
`/oidc/logout/`) continue to use mozilla-django-oidc's default views.
