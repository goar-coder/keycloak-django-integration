# Implementation Plan: D3 Django Service

**Branch**: `005-d3-django-service` | **Date**: 2026-06-26 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/005-d3-django-service/spec.md`

---

## Summary

Añadir un tercer servicio Django (D3) al stack existente. D3 es funcionalmente idéntico a D1
en cuanto a infraestructura y autenticación, pero con su propio cliente Keycloak (`d3-client`),
su propia base de datos (`d3_db`), su propio puerto (`8003`) y tres rutas protegidas por grupos
Keycloak (`d3:normas`, `d3:documentos`, `d3:leyes`) en lugar de las rutas de D1.
No incluye gestión de usuarios via Admin API (esa capacidad es exclusiva de D1).

---

## Technical Context

**Language/Version**: Python 3.12, Django 5.x

**Primary Dependencies**:
- `mozilla-django-oidc 4.x` — OIDC Authorization Code Flow con Keycloak
- `django-environ 0.11.x` — configuración via variables de entorno
- `psycopg2-binary 2.x` — adaptador PostgreSQL
- `whitenoise 6.x` — archivos estáticos
- `gunicorn 22.x` — servidor WSGI en producción
- `pytest 8.x` + `pytest-django 4.9.x` — suite de tests

**Storage**: PostgreSQL 16 — nueva base de datos `d3_db` con usuario `d3_user`. Sin tablas nuevas más allá del esquema estándar de D1 (`django_session`, `auth_user`, `accounts_userprofile`, `auth_group`).

**Testing**: pytest + pytest-django. Mínimo un test por vista (constitution requirement).

**Target Platform**: Linux Docker container, mismo Docker Compose stack que D1 y D2.

**Project Type**: Web application Django (views + templates). Sin DRF. Sin Admin API.

**Constraints**:
- Zero cambios observables en D1 y D2
- D3 no puede acceder a `d1_db` ni `d2_db` — usuario PostgreSQL aislado
- La configuración de Keycloak se codifica íntegramente en `realm-export.json`
- No hay hardcoded credentials — todo via `.env`

**Scale/Scope**: Dev/test environment. Misma escala que D1 y D2.

## Detailed Change Specifications

### 1. `postgres/init.sql` — añadir d3

```sql
CREATE USER d3_user WITH PASSWORD 'd3_pass' CREATEDB;
CREATE DATABASE d3_db OWNER d3_user;
REVOKE ALL ON DATABASE d3_db FROM PUBLIC;
GRANT ALL PRIVILEGES ON DATABASE d3_db TO d3_user;
```

### 2. `docker-compose.yml` — servicio d3

```yaml
d3:
  build: ./d3
  command: python manage.py runserver 0.0.0.0:8000
  volumes:
    - ./d3:/app
  environment:
    DJANGO_SETTINGS_MODULE: config.settings.development
    D3_SECRET_KEY: ${D3_SECRET_KEY}
    D3_DATABASE_URL: postgres://d3_user:${D3_DB_PASSWORD}@postgres:5432/d3_db
    KEYCLOAK_SERVER_URL: ${KEYCLOAK_SERVER_URL:-http://keycloak:8080}
    KEYCLOAK_PUBLIC_URL: ${KEYCLOAK_PUBLIC_URL:-http://localhost:8080}
    KEYCLOAK_REALM: ${KEYCLOAK_REALM:-app-realm}
    D3_OIDC_CLIENT_ID: ${D3_OIDC_CLIENT_ID:-d3-client}
    D3_OIDC_CLIENT_SECRET: ${D3_OIDC_CLIENT_SECRET}
    D3_REDIRECT_URI: ${D3_REDIRECT_URI:-http://localhost:8003/oidc/callback/}
    ALLOWED_HOSTS: localhost,d3
    DEBUG: "True"
  ports:
    - "8003:8000"
  depends_on:
    keycloak:
      condition: service_healthy
    postgres:
      condition: service_healthy
  healthcheck:
    test: ["CMD-SHELL", "curl -f http://localhost:8000/health/ || exit 1"]
    interval: 10s
    timeout: 5s
    retries: 5
    start_period: 20s
```

### 3. `keycloak/realm-export.json` — cinco adiciones

Ver `data-model.md` para los fragmentos JSON completos.

**3a. Grupo `d3:normas`** — nuevo grupo en `groups` del realm  
**3b. Grupo `d3:documentos`** — nuevo grupo en `groups` del realm  
**3c. Grupo `d3:leyes`** — nuevo grupo en `groups` del realm  
**3d. Cliente `d3-client`** — nuevo cliente confidential Authorization Code  
**3e. Rol `can-login` en `d3-client`** — rol en `roles.client.d3-client`  
**3f. AuthenticatorConfig `d3-client-role-cfg`** — `condUserRole: d3-client.can-login, negate: true`  
**3g. Flujos D3** — `D3 Browser Flow`, `d3-client-auth-methods`, `d3-client-forms`, `d3-client-otp`, `d3-client-role-check` (mirrors D1)

### 6. `d3/portal/views.py` — rutas protegidas

```python
from django.contrib.auth.mixins import LoginRequiredMixin
from django.views.generic import TemplateView
from accounts.decorators import require_groups

class HomeView(LoginRequiredMixin, TemplateView):
    template_name = 'portal/home.html'

class NormasView(LoginRequiredMixin, TemplateView):
    template_name = 'portal/normas.html'

    @method_decorator(require_groups(['d3:normas']))
    def dispatch(self, *args, **kwargs): ...

class DocumentosView(LoginRequiredMixin, TemplateView):
    template_name = 'portal/documentos.html'

    @method_decorator(require_groups(['d3:documentos']))
    def dispatch(self, *args, **kwargs): ...

class LeyesView(LoginRequiredMixin, TemplateView):
    template_name = 'portal/leyes.html'

    @method_decorator(require_groups(['d3:leyes']))
    def dispatch(self, *args, **kwargs): ...

class AccessDeniedView(TemplateView):
    template_name = 'portal/access_denied.html'
```
